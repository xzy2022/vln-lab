# 日报合成 skill

## 文件位置

- 输入：`Notes/10_Projects/工作报告/素材/MMDD.md`
- 输出：`Notes/10_Projects/工作报告/每日报告/MMDD.md`（MMDD 为当天日期，如 `0416.md`）

## 职责

读取今日素材文件，按三维度 × 多领域合成一份完整日报，输出到 `Notes/10_Projects/工作报告/每日报告/MMDD.md`。

## 三维度 × 多领域

三个维度对任何领域平等适用——不再有"科研专属"账户：

| 维度 | 积累什么 | 日报中何时写入 |
|---|---|---|
| 认知 | 对领域的理解加深 | 有新的理解就写 |
| 证据 | 实际产出和可检验结果 | 有产出就写 |
| 叙事 | 把认知组织成完整表达 | 只在真有进展时写，不硬填 |

当前活跃领域（与 record skill 同步）：

- **VLN研究** / **AI小说创作** / **健身** / **音乐** / **投资** / **AI工具与方法**

## 设计原则

- **领域平权**：所有领域在日报中地位相同，不因内容多少、是否科研而区别对待。
- **透明可纠偏**：每条认知必须标注来源（素材文件的话题标题）。不标来源的条目不写入。
- **领域逐节展开**：日报按领域分节，每个领域一张表，每一条素材一行。今天没有素材的领域不出现。
- **最小可用**：日报只有领域分节 + Next 节。不做自动关联、不做跨日追踪。
- **校验优先于生成**：合成后输出自检清单，让用户能一眼判断日报是否可靠。
- **用户在闭环里**：你负责聚合、去重、格式化。不替你筛选——素材里有的就进入日报。筛选发生在 record 阶段。

## 执行流程

1. 读取 `Notes/10_Projects/工作报告/素材/MMDD.md`（今日文件）。
2. 如果文件不存在或为空：直接告诉用户"今天没有素材记录"，不强行生成空洞日报。
3. 按领域 → 维度整理每条素材：
   - 同一领域同一话题下高度相似的条目合并为一条
   - 每条素材作为独立一行，不因类型相同而强行合并
   - 只保留今天素材中实际出现了的领域
4. 生成日报，写入 `Notes/10_Projects/工作报告/每日报告/MMDD.md`。
5. 输出日报内容到对话，附带自检清单。

## 日报格式

```markdown
# MMDD 日报

## <领域1>

| 类型 | 说明 | 链接资料 |
| --- | --- | --- |
| 认知 | ... | → [[10_Projects/工作报告/素材/MMDD#话题A]] |
| 证据 | ... | → [[10_Projects/工作报告/素材/MMDD#话题B]] |
| 叙事 | ... | → [[10_Projects/工作报告/素材/MMDD#话题C]] |

## <领域2>

| 类型 | 说明 | 链接资料 |
| --- | --- | --- |
| 认知 | ... | → [[10_Projects/工作报告/素材/MMDD#话题D]] |
```

规则：
- 每个领域一节 `#<领域名>`，只出现今天有素材的领域。
- 每个领域内一张三列表：**类型**（认知/证据/叙事）、**说明**、**链接资料**。
- 每条素材一行，按素材中的顺序排列。多条同类型不合并——保留独立条目。
- **链接资料**列追溯回素材文件：`10_Projects/工作报告/素材/MMDD#话题名`。只使用一个`#`,不写出`.md`后缀,且中间没有空格.
- 某领域今天只有一两条也正常——不硬凑三个维度。
- 叙事类型只在有真实进展时写入，没有则不写该行。
- Wiki-link 格式：`[[10_Projects/工作报告/.../MMDD|MMDD]]`，路径相对于 vault 根（即 `Notes/`）。

## 自检清单

日报输出后，在末尾附上：

```
## 自检

- [ ] 每条内容是否标注了来源（领域 + 话题）？
- [ ] 各领域之间是否有重复条目？
- [ ] 证据类型的行是否都是今天的实际产出（不是历史记录）？
- [ ] 叙事类型是否只在有真实进展时填写，而非硬凑？
- [ ] 是否有"读起来合理"但缺乏实质内容的空洞条目？
- [ ] Next 节中的待办是否都从今日认知中引出？
- [ ] 日报中只包含今天有素材的领域（没有凭空出现的领域节）？
```

## 边界

- 仅从素材文件合成，不做 history 兜底。
- 不自动生成周报。
- 不跨日聚合——那是周报 skill 的职责。
