# 速记 skill

## 文件位置

素材文件：`Notes/10_Projects/工作报告/素材/MMDD.md`（MMDD 为当天日期，如 `0427.md`）

## 职责

从当前对话中提取认知增量，追加到今日素材文件。

## 三维度 × 多领域

三个维度对任何领域平等适用：

| 维度 | 积累什么 | 典型产物 |
|---|---|---|
| 认知 | 对领域的理解加深 | 概念梳理、原理理解、行业认知 |
| 证据 | 实际产出和可检验结果 | 实验数据、训练记录、创作字数、交易记录 |
| 叙事 | 把认知组织成完整表达 | 论文段落、情节大纲、投资笔记、教程文档 |

当前活跃领域（可按需扩展）：

- **VLN研究** — 视觉语言导航的模型、实验、论文
- **AI小说创作** — AI 辅助网文写作的方法和产出
- **健身** — 训练计划、动作理解、身体数据
- **音乐** — 乐器练习、乐理理解、创作
- **投资** — 市场认知、交易记录、策略笔记
- **AI工具与方法** — skill 写法、工作流设计、AI 时代能力思考

## 设计原则

- **用户在闭环里**：你向用户提议要记什么，用户确认后才写入。不替用户做内容判断。
- **只存认知增量**：不是对话转录。每条 1-2 句，写的是"我现在对这个问题的理解发生了什么变化"。
- **来源可追溯**：每条末尾标注 `→ 当前对话`。日报合成时会用这个做交叉校验。
- **透明**：向用户展示你提议追加的内容、所属领域、维度和话题，让用户能一眼校验。

## 执行流程

1. 读取 `Notes/10_Projects/工作报告/素材/MMDD.md`,确认今日已经有哪些素材（不存在则创建）。
2. 回看当前对话中最近的讨论,识别：**属于哪个领域？话题是什么？新的认知增量是什么？属于哪个维度？**,优先考虑向已有的素材进行同类匹配。
3. 向用户展示提议：
   ```
   我理解你想记录：
     领域：VLN研究 / AI小说创作 / 健身 / 音乐 / 投资 / AI工具与方法
     维度：认知 / 证据 / 叙事
     话题：<话题名>
     内容：<1-2 句提炼>
   对吗？需要修改吗？
   ```
4. 用户确认后,在`Notes/10_Projects/工作报告/素材/MMDD.md`中按领域和话题组织：
   - 文件第一级标题：`# 今日素材 MMDD`
   - 第二级：`## <领域名>`
   - 第三级：`### <话题名>`（该领域下的具体话题）
   - 定位到对应领域节 → 对应话题节 → 追加一条 `- (HH:MM, <维度>) <内容> → 当前对话`
   - 如果该领域节不存在，新建
   - 如果该话题节不存在，在该领域节下新建
5. 汇报：写入了哪个领域、哪个话题、追加了几条。

## 文件格式

素材文件按日组织，领域第一级，话题第二级：

```markdown
# 今日素材 MMDD

## VLN研究

### stop 问题
- (14:30, 认知) 确认了 stop 时机是 SR-OSR gap 的主因，SOON 上尤其明显 → 当前对话
- (16:10, 证据) 产出了 oracle_success vs final_success 的对比表 → 当前对话

### 实验流程设计
- (15:00, 叙事) 形成了 eval-only 流程的完整方案草图 → 当前对话

## AI工具与方法

### skill 设计
- (11:00, 认知) 三维度 × 多领域的矩阵结构比领域专属账户更合理 → 当前对话
```

## 边界

- 如果领域或话题不清晰，先问用户怎么归类再写。
- 如果今日素材文件已有当天的标题，追加内容而不是覆盖。
- Wiki-link 格式：`[[10_Projects/工作报告/.../MMDD|MMDD]]`，路径相对于 vault 根（即 `Notes/`）。
