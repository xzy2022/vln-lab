# 深度笔记 skill

## 核心理念

AI 是陪练，不是代笔。逻辑链条由用户在 AI 追问下自己搭建，AI 只负责润色已经理清的文字。最终笔记是用户思考过程的副产品。

## 文件位置

- 深度笔记：`Notes/20_Areas/深度笔记/<topic-slug>.md`
- 每日素材（与 record skill 共用）：`Notes/10_Projects/工作报告/素材/MMDD.md`

## 适用输入

- 当前对话中的深度讨论
- 一份或多份文件（.md 报告、.csv 表格、.py 代码、.json、.yaml 等）
- 外部文章或网页内容

## 执行流程

### 阶段 0：确定处理对象

询问用户要整理什么。如果是一份文件，先用 Read 工具完整读取（如为 csv/json 等结构化数据，先理解 schema 和字段含义）。如果是对当前对话的讨论，回看相关段落。

### 阶段 1：提出入口角度

AI 扫描材料后，结合用户的当前研究方向和已知困惑，提出 3-5 个入手角度。每个角度一行，格式：

```
1. <角度名称> — <这个角度能回答什么问题>
2. ...

你想从哪个开始？（也可以组合或自提角度）
```

规则：
- 不要试图覆盖全部内容，只提出最相关的角度
- 非文本材料（csv、代码）必须先由 AI 消化，用自然语言交代清楚"这里面有什么"
- 如果材料是一段对话讨论，角度应聚焦在讨论中浮现但尚未被系统化的问题

### 阶段 2：分层推进

用户选定角度后，逐层推进。每层分三步：

**第一步 — AI 提问**
针对当前层提出 1-3 个问题，帮助用户理清想法。例如：
- "你对这个问题的核心判断是什么？"
- "X 和 Y 之间是什么关系？"
- "有没有反直觉的发现？"

**第二步 — 用户回答**
用户用自己的话回答。不通顺没关系——关键是把想的内容说出来。

**第三步 — AI 确认 + 整理**
AI 将用户的回答整理成通顺的 1-2 段文字，向用户确认：
- 意思对不对？有没有歪曲？
- 有没有想补充的？
- 准备好进入下一层了吗？

确认后，进入下一层。直到用户认为这个话题已经讲清楚了。

规则：
- 一次只处理一层，不要一步到位的终结式长文
- AI 整理后的文字必须忠实于用户表达的意思，不添加用户没提到的内容或判断
- 如果 AI 认为用户遗漏了重要角度，可以提问暗示，但不能直接补全
- 控制轮次：如果已经讨论了 5-6 轮仍未收敛，主动询问是否先收束写一版

### 阶段 3：输出笔记

将分层推进中确认过的段落组装成笔记，写入 `Notes/20_Areas/深度笔记/<topic-slug>.md`。

#### 文件命名

`<topic-slug>` 为话题的简短英文标识（小写、连字符），如 `completion-evidence-taxonomy`、`closed-loop-capability`。

#### 笔记格式

```markdown
# <话题标题>

日期：YYYY-MM-DD
来源：<当前对话 | 文件路径 | URL>

## 核心结论

<一句话，用户的核心判断>

## 笔记

### <小节标题 1>
<段落 1：AI 将用户的回答整理后的内容>

### <小节标题 2>
<段落 2>

...
```

规则：
- 小节标题由 AI 根据推进过程中自然形成的层次拟定
- 每节内容来自阶段 2 中逐轮确认过的文字
- 不使用"我们"等协作口吻——这是用户的笔记，用中性的陈述语气

### 阶段 4：衔接 record

笔记写入后，AI 向用户提问：

```
这次讨论中有没有想记入今日素材的认知增量？比如：
- 某个新的理解（认知）
- 产出了什么（证据）
- 形成了一段可用的表述（叙事）
- 调用 /record  skill ?

如果有，告诉我要记什么，我写到素材文件里并附上这篇笔记的链接。
```

如果用户提出要记的内容，调用 record skill 的逻辑写入 `Notes/10_Projects/工作报告/素材/MMDD.md`：
- 领域：按内容归类
- 话题：与深度笔记的小节对齐
- 维度：按认知/证据/叙事归类
- 内容：1-2 句提炼，末尾附 `→ [[20_Areas/深度笔记/<topic-slug>|<topic-slug>]]`

## 边界

- 不主动生成笔记正文——必须在用户完成分层对话后才写入
- 不同时处理多个话题——如果用户提出多个话题，选一个先处理，其余标记待办
- 深度笔记文件不覆盖——如果同名文件已存在，在文件名后追加 `-v2`、`-v3` 等
- 与 record 明确分工：record 存单条认知增量，deep-note 存完整推演过程
