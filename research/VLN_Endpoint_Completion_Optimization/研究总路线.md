# VLN Endpoint Completion Optimization：研究总路线

本文档是本研究方向的唯一总路线入口，用来确定研究问题、阶段划分、协议冻结原则和文档更新纪律。

每个阶段的完整实验结果、表格、失败分析和结论应该单独写入阶段成果文档；本文档只保留“路线级事实”和“当前决策状态”。后续每次更新文档时只更新一处：如果改的是总路线，只改本文档；如果补的是某个阶段结果，只改对应阶段文档。

## 0. 文档纪律

### 0.1 单一事实源

```text
总路线、阶段定义、冻结原则：只写在 研究总路线.md
阶段实验结果、命令、表格、产物路径：只写在对应阶段成果 md
临时想法、对话草稿、未审核推断：不直接混入冻结文档
```

如果某个结论需要同时出现在多个文档中，只保留一个 canonical 版本，其他地方只链接或引用，不复制展开。

### 0.2 更新规则

每次文档更新必须满足：

1. 只更新一个 markdown 文件。
2. 明确本次更新属于“总路线调整”还是“阶段成果补充”。
3. 没有人工审核前，新增判断一律标记为 `待审核`。
4. `val_unseen` 结果只能作为冻结协议后的最终报告，不能反向修改阈值、tau 或模型选择。

## 1. 核心研究问题

当前研究不直接做 full RL，也不重训 SAME 主导航器，而是研究：

```text
SAME 已经走过但没有正确 STOP 的 endpoint failure，
能否通过离线诊断、保守 reranking 和 learned endpoint preference，
在低 harm 的前提下恢复一部分 oracle gap？
```

当前主线暂定名：

```text
Evidence-Adaptive Completion Optimization for VLN
```

更具体地说：

```text
用 episode-level diagnostics、endpoint preference 和 category-aware routing，
在不重训主导航器的情况下减少 SAME 的 recoverable failure。
```

## 2. 研究边界

### 2.1 做什么

```text
分析 final STOP / endpoint failure
估计 SAME 已访问 endpoint 的可恢复上界
测试 rule-only endpoint reranking 是否可靠
构造严格离线 endpoint learning 闭环
学习 conservative gate + ranker
在冻结配置下一次性报告 val_unseen
```

### 2.2 不做什么

```text
不做在线 RL rollout
不在 val_unseen 上调参
不把 oracle distance 用作训练/推理特征
不把 smoke score 当 learned method
不让多个文档同时维护同一张结果表
```

## 6. 阶段 4 的统一口径

阶段 4 统一命名为：

```text
离线 endpoint learning 闭环
```

旧称 `轻量 endpoint scorer / preference loss` 不再作为阶段名称使用；它只描述 4.3 / 4.6 中的一类模型或 loss。

阶段 4 的目标：

```text
用固定协议构造 endpoint candidate group，
在 train/dev 上学习和冻结 gate、ranker、tau，
然后一次性报告 val_unseen，
验证 learned endpoint preference 是否比 rule-only heuristic 更稳。
```

### 6.1 子阶段

| 子阶段 | 目标 | 产物 | 是否训练 |
| --- | --- | --- | --- |
| 4.1 数据与评测闭环 | 固定 candidate / label / pair / tau-curve 协议 | dataset builder + evaluator | 否 |
| 4.2 gate-only baseline | 学“何时不要动 final” | gate baseline report | 只训 gate |
| 4.3 endpoint ranker baseline | 学候选 endpoint 排序 | ranker + inference report | 是 |
| 4.4 gate + ranker 冻结配置 | 固定 learned method 的推理策略 | frozen config + dev report | 不新增训练 |
| 4.5 ranker 诊断收束 | 定位 CE ranker 的 group top1 失败原因 | diagnostics report | 否 |
| 4.6 endpoint preference objective 修正 | 比 SPL-aware pairwise / group-listwise / final-preserving / actionable gate | ablation table | 是 |
| 4.7 dev 止损与 val_unseen 一次性报告 | 达标后冻结配置并最终验证；不达标则收束为 negative result | final report 或 negative result | 否 |

### 6.2 协议冻结

4.1 后冻结：

```text
candidate = all trajectory steps
should_rerank = final_fail && oracle_success
pair types = success>fail, better_SPL>worse_SPL, final_success final>bad_nonfinal
target_scope = official
```

训练参数只在 train/dev 上定：

```text
model type
hidden dim
learning rate
loss weights
class weights
pair sampling ratio
```

推理参数只在 dev 上定：

```text

tau
allow_change_final
top-k candidate restriction
```

`val_unseen` 只用于冻结配置后的最终报告。

## 7. 成功标准

第 4 阶段的最低成功标准：

```text
val_unseen delta_SR > 0
harm_rate <= 1pp
SPL 不下降
```

更强标准：

```text
recover >= 10% oracle gap
R2R val_unseen SR 提升 >= 0.85pp
harm_rate <= 1pp
```

如果第 4 阶段没有达到最低标准，不进入 NavNuances router 主实验，只保留为 negative result 或回到 failure category analysis。

4.6 后进入 val_unseen 前，dev 至少满足：

```text
weighted delta_SR >= +0.20pp
weighted delta_SPL >= 0
harm_rate <= 1pp
max_dataset_harm_rate <= 1pp
CVDN delta_SR >= -0.10pp
recovered / harmed > 1
```
